// about

A QA engineer who likes fixing the process, not just the bugs

I've spent the last three years finding the things nobody wants to find — race conditions in real-time syncs, broken auth flows, push notifications that quietly fail on Android but work fine everywhere else. My background covers manual testing, API validation, automation, security testing, and performance work, and I'm just as comfortable writing a test plan as I am explaining to a client why a release slipped by a day.

What I actually enjoy is the process side of things: building regression suites that catch issues before they ship, setting up documentation that makes handovers painless, and mentoring junior testers so the whole team gets a little faster with every release.
